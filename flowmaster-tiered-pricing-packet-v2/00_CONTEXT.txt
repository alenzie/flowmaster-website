================================================================================
FLOWMASTER TIERED PRICING SYSTEM - PROJECT CONTEXT
================================================================================

Project: Arcade-Style Early Adopter Pricing with Real-Time Tier Progression
Site: https://www.flowmaster.live
Repos: flowmastersuite (backend/admin), flowmaster-site (Astro frontend)

--------------------------------------------------------------------------------
BUSINESS GOAL
--------------------------------------------------------------------------------

Create a transparent, gamified pricing system that rewards early adopters with 
lower prices. As more people buy, the price increases. This creates real 
urgency without fake scarcity—customers can see exactly how many keys remain 
at each price tier.

The UI should feel like selecting a car in an arcade racing game from the 
2000s: a horizontal carousel of selectable "cards" with the active tier 
floating, glowing, and animated while unavailable tiers are grayed out.

--------------------------------------------------------------------------------
TIER STRUCTURE
--------------------------------------------------------------------------------

| Tier | Keys      | Price | Internal Name | Framing                    |
|------|-----------|-------|---------------|----------------------------|
| 1    | 1-15      | $12   | pioneer       | "Basically a beta tester"  |
| 2    | 16-400    | $19   | early_adopter | "Early adopter deal"       |
| 3    | 401-2000  | $29   | believer      | "Still got in early"       |
| 4    | 2001+     | $39   | standard      | "Normal price"             |

Special: Tier 1 buyers receive a personalized digital message (~80 words) 
from Huskie in their delivery email as a thank-you for being first.

--------------------------------------------------------------------------------
CURRENT USER FLOW (NO PAYMENT)
--------------------------------------------------------------------------------

1. User receives invite link → /invite?code=XXXXX
2. JS calls validate-invite-link edge function
3. Success reveals a pre-created beta key + copy button + download link
4. User downloads installer separately

Scouts get keys via magic link signup flow.

--------------------------------------------------------------------------------
NEW USER FLOW (WITH PAYMENT)
--------------------------------------------------------------------------------

1. User visits /pricing
2. Sees arcade-style tier selector with current tier highlighted
3. Clicks "Get This Key" on current tier
4. Redirects to Stripe Checkout (hosted by Stripe)
5. Completes payment
6. Stripe webhook fires → key assigned in database
7. User redirected to /purchase-success?session_id=XXX
8. Success page reveals key + sends email
9. User downloads installer from /download
